Everything is wired through three building blocks: a first-stage model that compresses images into a latent space, a generator that learns the distribution in that space, and GenWrapper that connects the two automatically. First-stage models can optionally apply vision-foundation alignment (DINOv2, MAE, or BiomedCLIP) to make the latent space semantically structured — particularly valuable for medical imaging and multi-modal training.

First Stage Tokenizers & VAEs Continuous KL autoencoders, discrete VQ/LFQ/BSQ/FSQ stacks, token sequence models. Maps pixels → latents.
Generator Diffusion & AR Models Diffusion transformers (DiT, MDT, UViT), masked generators (MaskGIT, MAGE), continuous AR (MAR, RAR) and more.
GenWrapper Glue Layer Selects encode/decode routes from the tokenizer type and generator type. One line to wire any combination.

Attention mechanisms & PyTorch backends

Every model in MedLat uses torch.nn.functional.scaled_dot_product_attention (SDPA). PyTorch selects the most efficient kernel automatically — but you can override it.

FlashAttention

Fastest; O(N) memory. Requires CUDA, fp16/bf16, head-dim ≤ 128. Not supported for float additive masks (MDT relative-position bias).

Memory-efficient attention

CUDA, handles most dtypes and mask types including float additive bias. Good fallback when FlashAttention is unavailable.

Math (unfused)

Universal fallback — CPU, any dtype, any mask. Slower and higher memory, but always works.

Default behaviour

By default PyTorch auto-selects the best available backend. On a CUDA device with fp16/bf16 this is FlashAttention; on CPU it falls back to math.

Selecting a backend — PyTorch ≥ 2.3

# preferred API (PyTorch ≥ 2.3)
from torch.nn.attention import sdpa_kernel, SDPBackend

# FlashAttention only
with sdpa_kernel(SDPBackend.FLASH_ATTENTION):
    output = model(x)

# Memory-efficient only
with sdpa_kernel(SDPBackend.EFFICIENT_ATTENTION):
    output = model(x)

# Math (unfused) — always available
with sdpa_kernel(SDPBackend.MATH):
    output = model(x)

Selecting a backend — PyTorch 2.0 / 2.1

# legacy API (PyTorch 2.0 / 2.1)
with torch.backends.cuda.sdp_kernel(
    enable_flash=True, enable_math=False, enable_mem_efficient=False
):
    output = model(x)  # FlashAttention

# Disable FlashAttention entirely (useful for debugging or float masks)
torch.backends.cuda.enable_flash_sdp(False)
print(torch.backends.cuda.flash_sdp_enabled())  # False

Per-model notes

  • MDT — passes a float additive relative-position bias as attn_mask. FlashAttention does not support float masks; PyTorch automatically falls back to memory-efficient or math.
  • Taming GPT — uses is_causal=True during training and is_causal=False for KV-cache inference (when layer_past is supplied).
  • VMAE — SDPA is used by default; when return_attn_map=True is requested the model falls back to manual attention to expose the intermediate attention weights.
  • UViT / TiTok — use an ATTENTION_MODE flag; SDPA is the default mode and requires no further configuration.
  • Muse Attend — applies a non-standard scale (scale=8) for RMSNorm-ed queries/keys; this is forwarded directly to SDPA via the scale= argument.

2D vision & 3D medical data

Single int → square/cubic. Pass img_size=224 or img_size=64 and every model internally expands to (224,224) or (64,64,64) depending on dims. 

get_model("mar.b", img_size=224, dims=2)
get_model("dit.xl_2", img_size=64, dims=3)

Tuple → non-square / anisotropic. Pass per-axis sizes for rectangular images or anisotropic volumes. All arithmetic (seq_len, patchify, unpatchify) adjusts automatically. 

get_model("mar.b", img_size=(192,256), vae_stride=8)
get_model("dit.xl_2", img_size=(32,128,128), dims=3)

Latent alignment

A tokenizer trained on reconstruction alone produces a compact latent space — but one that is geometrically opaque. Nearby latent codes carry no semantic guarantee, which makes generation harder. MedLat implements the Vision-Foundation (VF) alignment technique from VA-VAE and extends it with BiomedCLIP support for medical imaging. 

Image ──► Encoder ──► z ──► Decoder ──► reconstruction loss
                       │
                       ▼   (training only — foundation model is frozen)
              AlignmentModule
          ┌──────────────────────────────────────────────────┐
          │  Frozen foundation model                         │
          │  DINOv2-L  /  MAE-L  /  BiomedCLIP              │
          │            │                                     │
          │            ▼  target features                    │
          │  Learnable projection head on z                  │
          │            │                                     │
          │            ▼  predicted features                 │
          │  VF loss = distmat_loss + cosine_loss            │
          └──────────────────────────────────────────────────┘
                       │
                       ▼
          total loss = recon + KL + VF

The result: latent codes that are semantically structured — nearby points in latent space correspond to semantically similar images. Downstream generators converge faster and produce higher-quality samples.

medvae uses BiomedCLIP as its frozen teacher — making it the recommended choice for any medical imaging task.

Add alignment to any tokenizer

# Natural images → DINOv2 alignment
from medlat.modules.alignments import VFFoundationAlignment

alignment = VFFoundationAlignment(
    latent_channels=16,
    foundation_type="dinov2",  # "mae" | "dinov2" | "biomedclip"
)
tok = get_model("continuous.aekl.f8_d16",
                img_size=256, alignment=alignment)

# Medical images → BiomedCLIP alignment
alignment_med = VFFoundationAlignment(
    latent_channels=16,
    foundation_type="biomedclip",
)
tok_med = get_model("continuous.aekl.f8_d16",
                    img_size=256, alignment=alignment_med)

The VF loss — two complementary objectives

vf_loss_1 — structure. The pairwise cosine-similarity matrix of projected latents should mirror that of the frozen teacher features (with a configurable distmat_margin slack). This preserves the relational geometry of the representation.

vf_loss_2 — direction. Each individual spatial location should be directionally consistent with its counterpart in the teacher's feature space (with a configurable cos_margin). This anchors per-location semantics.

Multi-modal tip: Training on heterogeneous modalities (e.g. knee MRI + brain MRI)? Aligning both to the same foundation model (BiomedCLIP) gives the generator a consistent semantic coordinate system across modalities. The frozen teacher handles cross-modality normalisation implicitly.

Zero-friction API

v0.1.4 ships a suite of developer-experience improvements that eliminate the most common friction points when wiring models together.

Stop guessing kwargs

suggest_generator_params(tokenizer) inspects any tokenizer and returns the exact kwargs its generator needs — no more hunting for in_channels or vae_stride by hand. 

tok = get_model("continuous.aekl.f8_d16", img_size=256)
gen = get_model("dit.xl_2", img_size=256, num_classes=1000,
                **suggest_generator_params(tok))
# → {'vae_stride': 8, 'in_channels': 16}

Inspect before you build

get_model_signature(name) uses Python's inspect module to show every parameter a builder accepts — always in sync, zero docs lag. 

get_model_signature("dit.xl_2")
# → {'img_size': '<required>',
#    'vae_stride': '<required>',
#    'in_channels': '<required>',
#    'num_classes': 10, 'learn_sigma': True, …}

Instant error diagnosis

GenWrapper validates the tokenizer–generator pair at construction time and tells you exactly how to fix the mismatch — no silent failures, no puzzling runtime crashes. 

GenWrapper(gen, tok)
# ValueError: Channel mismatch: DiT.in_channels=4
#   but AEKL.embed_dim=16.
#   → Rebuild with in_channels=16, or pass
#     **suggest_generator_params(tokenizer)

Discover schedulers like models

The new scheduler discovery API mirrors the model registry — list what's available, read full metadata, then create. DDPM, Flow Matching, and Self-Flow all in one factory. 

available_schedulers()
# → ('diffusion', 'flow', 'self_flow')
info = scheduler_info("self_flow")
print(info.samplers, info.optional_kwargs)
sched = create_scheduler("flow", path_type="Linear")

GenWrapper also has a clean __repr__ — print(wrapper) shows the routing mode, model class names, and current scale_factor at a glance. Full reference at API docs → GenWrapper.

Quick start

from medlat import (get_model, available_models, get_model_info,
                   get_model_signature, suggest_generator_params,
                   GenWrapper, create_scheduler, available_schedulers)

# ── Discover what's registered ────────────────────────────────────────
available_models()                              # all 200+ IDs
available_models("discrete.vq.")             # VQ-only
get_model_signature("dit.xl_2")               # see required kwargs first

# ── Pipeline 1: Continuous VAE + DiT — params inferred automatically ─
tok = get_model("continuous.aekl.f8_d16", img_size=224)
gen = get_model("dit.xl_2", img_size=224, num_classes=10,
                **suggest_generator_params(tok))  # → vae_stride=8, in_channels=16
wrapper = GenWrapper(gen, tok)
print(wrapper)  # routing · model names · scale_factor

# ── Pipeline 2: Continuous VAE + MAR ─────────────────────────────────
tok = get_model("continuous.vavae.f8_d32_dinov2", img_size=224)
gen = get_model("mar.b", img_size=224, class_num=10,
                **suggest_generator_params(tok))
wrapper = GenWrapper(gen, tok)
loss = wrapper(wrapper.vae_encode(images), y=labels)

# ── Pipeline 3: Discrete VQ + MaskGIT ────────────────────────────────
tok = get_model("discrete.vq.f8_d4_e16384", img_size=224)
gen = get_model("maskgit.b", img_size=224, num_classes=10,
                **suggest_generator_params(tok))
wrapper = GenWrapper(gen, tok)

# ── Schedulers — discover, inspect, create ────────────────────────────
available_schedulers()                          # ('diffusion', 'flow', 'self_flow')
sched = create_scheduler("self_flow", masking_strategy="complexity")
sched = create_scheduler("flow",     path_type="Linear", prediction="velocity")

First-stage models

Map images or volumes into compact representations. All families support dims=2 and dims=3. Use available_models("continuous."), available_models("discrete."), or available_models("token.") to list.

Continuous autoencoders (continuous.*)

AEKL LDM-style KL autoencoder. Conv encoder/decoder, 12 configs from f4_d3 to f32_d64. The workhorse for diffusion-based generation. → paper
MAISI MONAI MAISI 3D-friendly KL AE — continuous.maisi.f4_d3 → paper
MedVAE KL AE with BiomedCLIP alignment for medical imaging — continuous.medvae.f8_d16, f8_d32 → paper
VAVAE Vision-foundation-aligned KL AE. Variants aligned to DINOv2, MAE, or BiomedCLIP — 9 configs at f8/f16 × d16/d32/d64. → paper
DCAE EfficientViT DC-AE with very high compression ratios — continuous.dcae.f32c32, f64c128, f128c512 → paper
SoftVQ / WQVAE Continuous wrappers with soft or warped quantization — continuous.soft_vq.* (with optional DINOv2/BiomedCLIP alignment), continuous.wqvae.f8_d4_e16384 → paper

Discrete VAEs (discrete.*)

VQ-VAE Vector Quantization VQGAN-style conv encoder/decoder. 12 configs, f4–f16, codebooks up to 16 384. discrete.vq.* → VQGAN paper
LFQ Lookup-Free Quant. Implicit codebook, bit-width controlled. 15 configs, f4–f16, bits 10–18. discrete.lfq.* → MagViT-v2 paper
BSQ Binary Spherical Quant. Binary representation on the sphere. Same sweep as LFQ. discrete.bsq.* → BSQ paper
FSQ Finite Scalar Quant. Per-channel finite alphabet quantization. discrete.fsq.* → FSQ paper
SimVQ SimVQ Simplified VQ with codebook collapse prevention. discrete.simvq.* → SimVQ paper
RQVAE Residual Quant. VAE Multi-level residual codebooks. Returns concatenated indices. discrete.rqvae.* → RQ-VAE paper
QINCo QINCo family Improved nearest-code quantizers and residual variants. discrete.simple_qinco.*, discrete.qinco.* → QINCo paper
HCVQ Hybrid Conv/ViT VQ Residual, soft, grouped, SD-VQ, MS-RQ and VAE presets. discrete.hcvq.* → paper

Token sequence models (token.*)

TiTok Compact 1-D token sequences for generation. Multiple scale/length presets. token.titok.* → TiTok paper
MAETok MAE-style reconstruction tokenizer. token.maetok.* → MAETok paper
VMAE ViT/VideoMAE-style encoder tokenizer. token.vmae.* → VMAE paper
DeTok Scale-grid tokenizer (ss / sb / … / xlxl). token.detok.* → DeTok paper
SoftVQ · ViTA Differentiable soft VQ tokenizer and ViT-based reconstruction AE. token.softvq.*, token.vita.* → SoftVQ paper

Generators

Three families: discrete AR (paired with discrete tokenizers), continuous AR (continuous tokenizers), and diffusion (continuous tokenizers). All support dims=2 and dims=3; img_size and patch_size accept int or tuple.

Discrete autoregressive

MaskGIT maskgit.b / .l / .h Iterative masked token generation with cosine masking schedule. Standard interface: wrapper(z, y=labels) → loss. Works with any discrete tokenizer. → MaskGIT paper
MAGE mage.xs_4 … mage.l_16 Masked Autoencoder as Generative model — ViT encoder/decoder on token grids. 12 configs (XS/S/B/L × patch 4/8/16). ⚠️ patch size suffix must match tokenizer vae_stride. → MAGE paper
Taming GPT taming.gpt_b / .gpt_l / .gpt_h Autoregressive next-token prediction (GPT-style). Classic causal transformer. Call as generator(z[:-1], targets=z[1:]) for the AR loss. → Taming Transformers paper
MaskBit maskbit.s / .b / .l · maskbit.bert_s / .b / .l BERT-style masked generation. LFQBert variants for LFQ/BSQ tokenizers; Bert variants for VQ/FSQ tokenizers. Uses a custom masking training loop. → MaskBit paper

Continuous autoregressive

MAR mar.b / .l / .h Masked AutoRegressive model with diffusion loss on continuous tokens. Takes (B, C, H, W) latents. Supports both conditional (y=labels) and unconditional training. Standard interface: wrapper(z, y=labels). → MAR paper
RAR rar.b / .l / .xl / .h Recurrent AutoRegressive model with diffusion loss. Expects flattened sequence (B, seq_len, embed_dim). Use force_one_d_seq to bypass the built-in patch embedder. → RAR paper
FractalGen fractal.ar_64 · fractal.mar_64 · fractal.mar_base/large/huge_256 Hierarchical fractal AR: multi-level MAR or AR cascade that progressively refines predictions. Requires a custom hierarchical training loop. → FractalGen code

Non-autoregressive (diffusion)

DiT dit.s/b/l/xl × _1/_2/_4/_8 Diffusion Transformer — patchified continuous latents, adaLN class conditioning. 16 configs. Use with create_gaussian_diffusion. → DiT paper
MDT mdt.s/b/l/xl × _2/_4 Masked Diffusion Transformer — encoder-decoder with masked side branches. 8 configs. Strong scaling on smaller latent grids. → MDT paper
UViT uvit.small / .small_deep / .mid / .large / .huge U-Net + ViT hybrid diffusion model. Skip connections between encoder and decoder blocks. 5 configs. → UViT paper
LDM ldm.f1 … ldm.f16 Classic Latent Diffusion UNet. Multiple stride variants for different latent resolutions. → LDM paper
ADM adm.diffusion.{64,128,256,512}{C,U} Dhariwal–Nichol UNet diffusion + matching classifier heads. Resolution-specific configs. Also: adm.classifier.*. → ADM paper

Reading registry IDs

  • f{N} — spatial compression factor (f8 = 8× per axis)
  • d{N} — latent channels / embedding dimension
  • e{N} — codebook size for VQ
  • b{N} — bit width (LFQ, BSQ)
  • l{N} — levels (FSQ)
  • s/b/l/xl/h — model scale (small → huge)
  • _2/_4/_8 — patch size in generator name
# Decode these at a glance:
"continuous.aekl.f8_d16"     # AE-KL, 8×, 16 ch
"discrete.lfq.f16_d14_b14"   # LFQ, 16×, 14d, 14 bits
"discrete.vq.f4_d3_e8192"    # VQ, 4×, 3d, 8192 codes
"dit.xl_2"                   # DiT-XL, patch 2
"mage.b_8"                   # MAGE-Base, stride 8
"mar.h"                      # MAR-Huge

FAQ

What is MedLat?

A PyTorch library that registers first-stage models (tokenizers, VAEs, quantizers) and generators (diffusion, masked, autoregressive) under string IDs. Import medlat and call get_model("registry.id", **kwargs) to instantiate any of them.

Why "MedLat"?

Med = medical imaging, rigorous evaluation, latent representations you can trust. Lat nods to mad lad — bold, a bit cheeky, impossible to ignore. The library is clinical where it counts and fun to ship.

What generators are included?

Discrete AR: MaskGIT, MAGE, Taming GPT, MaskBit. Continuous AR: MAR, RAR, FractalGen. Diffusion: DiT, MDT, UViT, LDM, ADM. That's 11 generator families and 50+ registered configs.

How do I connect a tokenizer to a generator?

Use GenWrapper(generator, tokenizer). It inspects both models and automatically selects the right encode/decode path — continuous vs discrete first stage, autoregressive vs non-autoregressive generator.

2D images and 3D medical volumes?

Yes. Pass dims=2 for 2D (default) or dims=3 for volumetric data. img_size and patch_size accept a single int (square/cubic) or a per-axis tuple — all models normalise internally via to_ntuple.

Can I train unconditionally (no class labels)?

Yes. MAR and MaskGIT both support unconditional training — pass y=None and the model uses a learnable fake-class embedding. For conditional training, label_drop_prob applies classifier-free guidance dropout automatically during training.
