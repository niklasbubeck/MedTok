Med · Lat · One registry

Medical depth. Mad-lad code.

The name? Medical meets mad lad—a cool library that still takes imaging seriously.

MedLat (medlat) wires first-stage tokenizers and autoencoders to second-stage generators through one model registry: hundreds of IDs, one API—get_model, available_models, get_model_info. Every family is available for 2D computer vision and 3D volumetric medical data—switch with dims=2 vs dims=3 (and matching spatial sizes).

2D vision & 3D medical data

All registered architectures are available for standard 2D pipelines (natural images, 2D slices, projections) and for 3D volumetric imaging (e.g. CT/MRI-style volumes). The registry IDs are the same—use constructor arguments such as dims=2 or dims=3 plus consistent img_size / spatial layout when you call get_model(...).

Why “MedLat”?

Med is the mission: medical imaging, solid evaluation, and latent representations you can trust. Lat nods to mad lad—British slang for someone bold, a bit cheeky, impossible to ignore. Put them together and you get a framework that does not hide behind beige enterprise: it’s rigorous where it counts and fun to ship.

What this project contains

Typical workflow: train or load a first-stage model → attach a generator whose tensor geometry matches the tokenizer or VAE (num_tokens, embed_dim, seq_len, stride, etc.) → sample or train using the bundled notebooks. Choose planar or volumetric inputs up front via dims and spatial config.

2D & 3D One registry for 2D CV and 3D medical volumes—dims=2 vs dims=3 on the same model IDs.
First stage Maps images or volumes to compact representations: continuous latents, discrete codes, or token sequences.
Generator Models the distribution in that space—MaskGIT, MAR, DiT, LDM-style UNet, or ADM.
Utilities Gaussian diffusion helpers, ViT cores, foundation alignments, and GenWrapper for wiring.

Installation

# Editable install from repository root
pip install -e .

Core dependencies: PyTorch, NumPy, Einops, timm, OmegaConf, MONAI. Optional [dev] extras in pyproject.toml. Some quantizer paths may require additional packages (e.g. wavelets).

Quick start

from medlat import available_models, get_model, GenWrapper
from medlat import get_model_info

# Discover registered IDs
print(list(available_models()))

# Metadata: description, paper_url, code_url when set
print(get_model_info("continuous.aekl.f8_d16"))

# dims=2 for 2D CV, dims=3 for volumetric medical data (adjust img_size accordingly)
tokenizer = get_model("token.titok.s_128", img_size=128, dims=2)

generator = get_model(
    "maskgit.b",
    img_size=128,
    seq_len=tokenizer.num_latent_tokens,
    num_tokens=tokenizer.n_embed,
    in_channels=tokenizer.embed_dim,
)

wrapper = GenWrapper(generator, tokenizer)

GenWrapper selects encode/decode paths from discrete vs continuous first stages and autoregressive vs non-autoregressive generators (see medlat/modules/wrapper.py).

Registry ID conventions

  • f{N} — spatial downsampling factor at the bottleneck.
  • d{N} — latent width / embedding dimension (family-specific).
  • e{N} — codebook size for vector quantization.
  • l{N} / b{N} — levels or bit width (FSQ, LFQ, BSQ).
  • Scale tags — s / b / l / xl for depth in several families.

FAQ

Why is it called MedLat?

It’s a blend of medical (the science we care about) and mad lad (slang for someone bold and memorable). The library is meant to feel capable and a little irreverent—without cutting corners on imaging.

What is MedLat?

A PyTorch library that registers first-stage and generator architectures under string IDs. Import medlat and call get_model("registry.id", **kwargs).

2D computer vision and 3D medical volumes?

Yes. The same registry covers 2D pipelines (natural images, slices) and 3D volumetric data (e.g. CT/MRI-style volumes). Pass dims=2 or dims=3 (and matching img_size / spatial layout) when you build a model.

What is the Python package name?

The project is MedLat; import the library as import medlat (see pyproject.toml).
